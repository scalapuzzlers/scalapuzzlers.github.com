Too many dots

Contributed by Rogach
Source stackoverflow
Tested with Scala version 2.9.2

What does the following code do?

val L = 1.::(2.::(Nil))
  1. rints: 
    L: List[Double] = List(1.0, 2.0)
  2. Prints: 
    L: List[Int] = List(1, 2)
  3. Fails to compile
  4. Prints: 
    L: List[Int] = List(2, 1)

Explanation

Scala parser interprets "1." as "1.0" - a Double! Thus the expression becomes: 

1.0 :: 2.0 :: Nil
