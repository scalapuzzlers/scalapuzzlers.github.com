--- layout: puzzler title: All's Well That Folds Well author: Andrew Phillips sourceName: scala-lang mailing list sourceHref: https://groups.google.com/forum/?fromgroups=#!topic/scala-language/4h52vHuS7Qw scalaVersion: 2.10.0 puzzle: | val ints = List(1, 2, 3) val accumulator: List[Int] = Nil val leftReversed = (ints foldLeft accumulator) { (acc, elem) => elem :: acc } val rightReversed = (ints foldRight accumulator) { (elem, acc) => acc :+ elem } println(leftReversed) println(rightReversed) // ...and back println((leftReversed /: accumulator) { (acc, elem) => elem :: acc }) println((rightReversed :\ accumulator) { (elem, acc) => acc :+ elem }) possibleSolutions: a: code: | List(3, 2, 1) List(3, 2, 1) List(1, 2, 3) List(1, 2, 3) b: code: | List(3, 2, 1) List(3, 2, 1) List(3, 2, 1) List(3, 2, 1) c: code: | List(3, 2, 1) List(1, 2, 3) List(1, 2, 3) List(3, 2, 1) d: code: | List(3, 2, 1) List(3, 2, 1) List(3, 2, 1) List(1, 2, 3) solution: d explanation: |

Even though it's tempting to simply replace the "longhand" form of foldLeft and foldRight with their shorthand equivalents /: and :\, it's important to remember Scala's operator associativity rule, which states that operators ending in : (such as foldLeft's shorthand form) are right associative.

This means that (leftReversed /: accumulator) { ... } is actually equivalent to (accumulator foldLeft leftReversed) { ... }, not (leftReversed foldLeft accumulator) { ... } as intended. 

  println((accumulator /: leftReversed) { (acc, elem) => elem :: acc })
produces the desired result. Avoiding the potential for confusion by sticking to the longhand form is probably also worth considering.
