Captured by Closures

Contributed by Simon Schaefer
Source stackoverflow
Tested with Scala version 2.10.0

What is the result of executing the following code?

val values = Seq(100, 110, 120)
val funcs1 = collection.mutable.Buffer[() => Int]()
val funcs2 = collection.mutable.Buffer[() => Int]()
var j = 0
for (i <- 0 until values.length) {
 funcs1 += (() => values(i))
 funcs2 += (() => values(j))
 j += 1
}

funcs1.foreach { f1 => println(f1()) }
funcs2.foreach { f2 => println(f2()) }
  1. funcs1 prints 
    100, 110, 120
    whereas funcs2 throws an ArrayIndexOutOfBoundsException
  2. Both print 
    100, 110, 120
  3. The code does not compile
  4. funcs1 prints 
    100, 110, 120
    whereas funcs2 prints 
    120, 120, 120

Explanation

When one uses a var instead of a val the functions closes over a variable and not over a value. Because i is defined in the for-comprehension it is defined as a val. This means each time i is stored somewhere its value is copied - therefore it prints the expected result 100, 110, 120. As j changes inside the loop, all the three closures "see" the very same variable j and not a copy of it. Because after the loop j is 3 an ArrayIndexOutOfBoundsException is thrown. It is possible to avoid the exception by "fixing" the value to be returned: 

val values = Seq(100, 110, 120)
val funcs1 = collection.mutable.Buffer[() => Int]()
val funcs2 = collection.mutable.Buffer[() => Int]()
var j = 0
for (i <- 0 until values.length) {
 funcs1 += (() => values(i))
 val value = values(j)
 funcs2 += (() => value)
 j += 1
}
funcs1.foreach { f1 => println(f1()) }
funcs2.foreach { f2 => println(f2()) }
value's assignment is carried out immediately, and thus "captures" the intended element of values.
