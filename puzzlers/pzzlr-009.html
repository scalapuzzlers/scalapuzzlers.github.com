Map Comprehension

Contributed by Simon Schaefer
Source Stack Overflow
First tested with Scala version 2.10.0

What is the result of executing the following code?

val xs = Seq(Seq("a", "b", "c"), Seq("d", "e", "f"), Seq("g", "h"), Seq("i", "j", "k"))

val ys = for (Seq(x, y, z) <- xs) yield x + y + z
val zs = xs map { case Seq(x, y, z) => x + y + z }
  1. Evaluating ys and zs both result in a MatchError
  2. Both ys and zs evaluate to 
    Seq(abc, def, ijk)
  3. ys throws a MatchError, zs evaluates to 
    Seq(abc, def, ijk)
  4. ys evaluates to 
    Seq(abc, def, ijk)
    zs throws a MatchError

Explanation

Most people think that a for-yield-expression is directly translated to an equivalent call of map, but this is only correct if no pattern-matching is used! If one uses pattern-matching a filter is also applied. In the example above, the following code is generated: 

xs withFilter { case Seq(x, y, z) => true; case _ => false } map { case Seq(x, y, z) => x + y + z }
Be aware of this silent filter! It can result in nasty bugs.
