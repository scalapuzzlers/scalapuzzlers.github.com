For Each Step...

Contributed by Marconi Lanna
Source N/A
First tested with Scala version 2.11.6

What is the result of executing the following code?

val numberNames = Map(1 -> "one", 2 -> "two")

def pingpongIterable(numbers: Iterable[Int]) = numbers map {
    num => println("Ping-i " + num); numberNames(num)
  } foreach {
    name => println("Pong-i " + name)
  }

def pingpongTraversable(numbers: TraversableOnce[Int]) = numbers map {
    num => println("Ping-t " + num); numberNames(num)
  } foreach {
    name => println("Pong-t " + name)
  }

pingpongIterable(1 to 2)
pingpongTraversable(1 to 2)
  1. Prints: 
    Ping-i 1
Pong-i one
Ping-i 2
Pong-i two

Ping-t 1
Pong-t one
Ping-t 2
Pong-t two
  2. Prints: 
    Ping-i 1
Ping-i 2
Pong-i one
Pong-i two

Ping-t 1
Ping-t 2
Pong-t one
Pong-t two
  3. Prints: 
    Ping-i 1
Ping-i 2
Pong-i one
Pong-i two

Ping-t 1
Pong-t one
Ping-t 2
Pong-t two
  4. The first statement prints: 
    Ping-i 1
Pong-i one
Ping-i 2
Pong-i two
    and the second throws a runtime exception

Explanation

Method map on Iterable, which is inherited by the Range class, returns a new collection created by applying the provided function to each element in the original iterable. In pingpongIterable, each element of the range numbers is retrieved, and "Ping-i" printed for it, before the foreach block is reached.

The trait TraversableOnce, on the other hand, does not define a map method, so Range's implementation of map is never invoked. The reason map can be called on TraversableOnce is due to an implicit conversion to TraversableOnce.MonadOps. In pingpongTraversable, the invocation of numbers.map is transformed into code equivalent to MonadOps(numbers) map { ... }.

MonadOps.map is implemented by converting the traversable to an iterator and calling map on that. As per the Scaladoc for Iterator.map, this returns a new iterator which transforms every value produced by the orginal iterator by applying the specified function when a value is requested. In the case of pingpongTraversable, therefore, the "Ping-t" statements are printed only when successive elements of the collection are retrieved in the foreach block.
