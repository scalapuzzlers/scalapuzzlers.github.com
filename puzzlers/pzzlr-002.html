Match Me

Contributed by Andrew Phillips & Nermin Serifovic
Source stackoverflow
Tested with Scala version 2.9.2

What is the result of executing the following code?

val ij: (Int, Int) = ("3", "4")
val (i, j): (Int, Int) = ("3", "4")
  1. Prints: 
    ij: (3, 4)
i: 3
j: 4
  2. Both fail with compilation exceptions
  3. Both fail with runtime exceptions
  4. The first one fails with a compilation exception and the second one fails with a runtime exception

Explanation

The former is an assignment of a tuple which is not type correct. The latter is a pattern match which for which type information is erased, so it will compile (with a warning) but fail at runtime.
