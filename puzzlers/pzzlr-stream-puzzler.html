Stream Oddity

Contributed by Andrew Phillips
Source scala-language mailing list
First tested with Scala version 2.11.6

What is the result of executing the following code?

val nats: Stream[Int] = 1 #:: (nats map { _ + 1 })
val odds: Stream[Int] = 1 #:: (odds map { _ + 1 } filter { _ % 2 != 0 })

nats filter { _ % 2 != 0 } take 2 foreach println
odds take 2 foreach println
  1. Prints: 
    1
3
1
3
  2. Prints: 
    1
2
1
3
  3. The first statement prints: 
    1
3
    and the second throws a runtime exception
  4. The first statement throws a runtime exception and the second prints: 
    1
3

Explanation

The expression nats filter { _ % 2 != 0 } constructs a new stream that pulls elements from nats and passes on only those that match the filter. The second element of the filtered stream is thus the third element of nats, namely the value 3.

Successive elements of nats can always be calculated because the next element of nats is simply the current element, plus 1. In the case of the stream odds, however, the filter is part of the recursive definition of the stream. This means that only values that match the filter can be returned as elements of odds.

This prevents the stream from calculating its second element, i.e. the first element of the stream odds map { _ + 1 } filter { _ % 2 != 0 }. The first attempt, using 1, the first element of odds, does not match the filter. The next attempt tries to use the second element of odds – but that is the precisely the value we are trying to calculate.

The attempt to calculate the second element of odds thus ends up calling itself recursively, leading to an endless loop and eventual StackOverflowError.
