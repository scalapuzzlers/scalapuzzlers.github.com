Sequence match puzzler

Contributed by Scala Team @ Razorfish
Source N/A
First tested with Scala version 2.11.7

What is the result of executing the following code?

  import java.util.{List => JList, LinkedList}
  import scala.collection.JavaConversions._

  def listFromJava: JList[Int] = {
    val jlist = new LinkedList[Int]()
    jlist.add(1)
    jlist.add(2)
    jlist
  }

  def printHeadOrEmpty(s: Seq[_]) = s match {
     case hd :: _ => println(hd)
     case _ => println("Empty :-(")
  }

  printHeadOrEmpty(listFromJava)
  printHeadOrEmpty(listFromJava.toSeq)
  printHeadOrEmpty(Seq(listFromJava: _*))
  1. Prints 
    1
1
Nothing
  2. Prints 
    Nothing
Nothing
1
  3. Prints 
    Nothing
1
1
  4. Prints 
    Nothing
Nothing
Nothing

Explanation

The frenquently used constructor scala.collection.Seq is an interface with an apply method that returns a List[Int] instance by default when called so Seq(1,2) therefore matches the first case. The java conversion method that's invoked when we call toSeq on a Java list, or when the LinkedList is passed directly, returns a Buffer because this is the only conversion supported for Java lists which doesn't match the first case (only a Scala list have :: defined on them). To catch all sequences we should used the +: for the matching case.
